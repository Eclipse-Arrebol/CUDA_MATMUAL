```
└── Makefile          # 分离编译,bench + verify 两个 target,verify 链 -lcublas

```

### 关键工程约定
- **B 路线**:所有 kernel 编进一个 bin,benchmark/verify 各自维护注册表 `g_kernels[]` / `g_names[]`,按 id 分发。
- **绝不 `#include .cu 文件**(踩过坑:导致 multiple definition)。跨文件只靠 kernels.h 的声明。
- 每个函数定义只能存在一处;头文件只放声明、宏、typedef。
- 调用约定:`./bench <id> M N K`、`./verify <id> M N K`,无参跑默认 1024³ id=0。
- 编译:`-arch=sm_75 -I include -O3`,改头文件需 `make clean`(Makefile 已加 common.h 依赖)。

### 注册表当前内容
- benchmark.cu: `{launch_dummy_kernel, launch_naive_kernel, launch_smem_kernel, launch_blocktiling_kernel}` → id 0/1/2/3
- verify.cu: `{launch_cublas_ref, launch_naive_kernel, launch_smem_kernel, launch_blocktiling_kernel}` → id 0/1/2/3
  - **注意:两个文件的 id 映射不同!** verify 的 id=0 是 cublas reference。用 verify 时 id=1 才是 naive。

### verify 判据
- 相对误差为主:`|mine-ref| / (|ref|+1e-5) < 1e-2`
- cuBLAS 调用用列主序技巧:传参时 A/B 交换、用 `N,M,K` 顺序、leading dim 见 verify.cu
- 健康标志:小尺寸(256)应出现非零小误差(~1e-3),证明 verify 有鉴别力;大尺寸偶现 0 属正常

> (相邻 thread 读 B 连续、读 A 广播,DRAM 仅 21%),coalescing 不是瓶颈,故直接上 SMEM。
> 这是一个有依据的决策点,portfolio 里如实记录。

### Kernel 3(1D Block Tiling)配置
- `BM=64, BN=64, BK=8, TM=8`;block = (64,8) = 512 线程;grid = CEIL_DIV(N,64)×CEIL_DIV(M,64)
- 核心:一个 thread 算同列连续 TM=8 个 C 元素,B 值读一次复用 8 次
- **三层独立索引映射**(易错点,已掌握):
  - 搬 As[BM][BK]:行 `tid/BK`、列 `tid%BK`
  - 搬 Bs[BK][BN]:行 `tid/BN`、列 `tid%BN`
  - 算 C:列 `tid%BN`、行段 `tid/BN`,行偏移 `(tid/BN)*TM + i`
  - 教训:三套坐标用不同变量名,别复用 ty/tx

## 四、关键技术认知(已内化)

1. **ncu 的 "Memory Throughput" 取内存子系统最忙的一级**,不等于 DRAM。naive 显示 82% 其实是 L1/TEX 忙,DRAM 只 21% → 瓶颈不是显存带宽。
2. **shared memory 和 L1 在同一块物理 SRAM**:ncu 的 L1/TEX throughput 把 shared 访问也算进去。所以 smem 版 L1/TEX 不降反升(84%→87%),因为 shared 访问占满了这块 SRAM。
3. **SMEM 版只快 29% 的原因**:数据搬进 shared 但每 thread 只算 1 元素,shared 访问比仍是 ~2:1,SRAM 仍是瓶颈。需要寄存器复用(tiling)才能真正卸载。
4. **三级数据流 global → shared → register**:shared 解决 block 级复用,register 解决 thread 级复用,两层叠加。
5. **tile 大小(BM×BN)与线程排布(blockDim)解耦**:一个 thread 算多元素后,`blockIdx.x*BN` 取代 `blockDim.x*blockIdx.x` 当列起点。
6. **测试尺寸影响结论**:小尺寸 + 大 L2 会掩盖访存瓶颈(4090 上 naive 数字诡异稳定即此故)。要用大尺寸暴露真瓶颈。
7. **行主序索引命门**:`B[行][列] = 行*N+列`(B 是 K×N,乘 N);`A[行][列]=行*K+列`(A 是 M×K,乘 K)。两个矩阵每行宽度不同。
