
## 1、先看大局观
``` text
ncu --set basic -k double_buffer_kernel --launch-count 1 \
  ./bench 11 4096 4096 4096
```

| 指标                 | 看什么                              |
| ------------------ | -------------------------------- |
| Duration           | kernel 实际跑多久                     |
| Compute Throughput | SM 计算管线忙不忙                       |
| Memory Throughput  | memory 系统整体忙不忙                   |
| DRAM Throughput    | 显存带宽忙不忙                          |
| L1/TEX Throughput  | L1/shared/TEX 片上 memory path 忙不忙 |
| L2 Throughput      | L2 cache 路径忙不忙                   |

## 2、再看延迟
``` text
ncu -k double_buffer_kernel --launch-count 1 \
  --metrics \
smsp__average_warps_issue_stalled_long_scoreboard_per_issue_active.ratio,\
smsp__average_warps_issue_stalled_short_scoreboard_per_issue_active.ratio,\
smsp__average_warps_issue_stalled_math_pipe_throttle_per_issue_active.ratio,\
smsp__average_warps_issue_stalled_mio_throttle_per_issue_active.ratio,\
smsp__average_warps_issue_stalled_barrier_per_issue_active.ratio \
  ./bench 11 4096 4096 4096
```

| 指标                                       | 说明                                    |
| ---------------------------------------- | ------------------------------------- |
| Warp Cycles Per Issued Instruction       | 单个 warp 平均隔多少 cycle 才能 issue 指令       |
| Warp Cycles Per Executed Instruction     | 单个 warp 平均隔多少 cycle 才执行指令             |
| Avg. Active Threads Per Warp             | 每条 warp 指令平均活跃线程数                     |
| Avg. Not Predicated Off Threads Per Warp | 没被 predicate 屏蔽的线程数                   |
| long_scoreboard                          | 等 global / local / texture 路径 load 数据 |
| math_pipe_throttle                       | math pipe 忙，数学指令发不进去                  |
| mio_throttle                             | MIO 队列 / 管线忙                          |
| short_scoreboard                         | 等 shared memory / MIO 结果              |

``` text
 ncu --section SchedulerStats -k regex:"naive" --launch-count 1 ./bench 1 4096 4096 4096
```

| 指标                           | 含义                            |
| ---------------------------- | ----------------------------- |
| Active Warps Per Scheduler   | 每个 scheduler 上有多少 active warp |
| Eligible Warps Per Scheduler | 每个 scheduler 上有多少 ready warp  |
| One or More Eligible         | 有至少一个 ready warp 的周期比例        |
| No Eligible                  | 没有任何 ready warp 的周期比例         |
| Issued Warp Per Scheduler    | 每周期实际发射多少 warp 指令             |

## 3.再看访存合并，blank conflict
``` text
ncu --metrics \
l1tex__average_t_sectors_per_request_pipe_lsu_mem_global_op_ld.ratio,\
l1tex__average_t_sectors_per_request_pipe_lsu_mem_global_op_st.ratio \
-k regex:"naive" --launch-count 1 ./bench 1 4096 4096 4096
```
ld就是读，value的单位是sector，一个sector是32byte，32 个线程连续写float的话其实就是4x32=128个byte，其实就是4sectors，也就是说一次ld指令最好就是4，st就是写和ld同理


``` text
ncu --metrics \
l1tex__data_bank_conflicts_pipe_lsu_mem_shared_op_ld.sum,\
l1tex__data_bank_conflicts_pipe_lsu_mem_shared_op_st.sum \
-k regex:"naive" --launch-count 1 ./bench 1 4096 4096 4096
```
这个就是看有没有读和写的blank conflict

ncu --section Occupancy \
