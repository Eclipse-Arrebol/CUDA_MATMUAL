核心想法：把 C 矩阵切成一个个小方块（tile），一个 block 负责算一个 C 的 tile。要算这个 C tile，需要 A 的对应几行、B 的对应几列。但这些行列太长（K=4096），一次塞不进 shared memory。所以再把 K 维也切块，分多步：每步把 A 的一小块和 B 的一小块搬进 shared，block 内所有 thread 从 shared 里取数做部分累加，然后搬下一块，累加，直到 K 走完。
```
(base) hp@Miliar:~/CUDA_MATMUAL$ ./bench 2 4096 4096 4096
smem_kernel: M=4096 N=4096 K=4096, time=214.8767 ms, GFLOPS=639.62
```

## 怎么想到这个算法的
其实看ncu的报告发现memory的压力还是很大的，这是因为A，B的访问元素是十分的频繁的，那有没有一种方法能够减少内存的访问呢，那就是share_memory,但是怎么使用share_memory呢，其实还是把问题看成一个矩阵乘法，只不过之前是一个元素，现在变成的一块元素，这一块就是share_memory，还是一个thread对应一个元素吗，是的，那现在就变成了thread自己所在的share_memory对应的元素读进来，那原来的对每个元素的遍历其实就变成了对每一块元素的遍历，但是遍历完每一块元素是不是要对每一块元素内部的行或者列做一个累加，就得到了最后的结果

其实它给人的感觉就是把一个循环拆成了两个循环
